# 정적 자산 최적화 리포트

## 1. 대용량 정적 파일 점검 결과
- 검사 범위: `public/`, `assets/`
- 임계치: 5 MB
- 결과: **5 MB 이상 파일 없음**

| 파일 경로 | 크기(MB) | 코드/템플릿 참조 여부 | 조치 |
| --- | --- | --- | --- |
| (해당 없음) | - | - | - |

> 참고: 상위 용량 파일 중 최대는 `public/sw.js` (약 0.0 MB)로 최적화 필요 없음.

## 2. 최적화 가이드
- 아이콘/일러스트: 현재 대용량 파일이 없어 SVG 전환 필요 없음. 향후 추가 시 SVG 우선 검토 권장.
- 사진류: 변환 대상 없음. 추후 5 MB 이상 이미지 추가 시 WebP 또는 JPG(품질 80~85)로 재인코딩 권장.
- 경로/파일명 유지 조건의 치환 대상 없음.
- 미사용 자산: 5 MB 이상 자산이 없어 제거 제안 불필요.

## 3. 예상 용량 절감량
- 현재 기준 예상 절감량: **0 MB**

## 4. 후속 조치 제안
- 향후 대용량 정적 파일 추가 시, 배포 전 동일한 임계치 검사(5 MB+)를 자동화하여 리포트 생성.
- assets/_backup/ 구조 도입은 실제 변환 대상 발생 시점에 맞춰 적용.
- 정기적으로 Lighthouse/웹 나열 검사로 이미지 최적화 상태 점검 권장.
