# 의존성 정비 리포트

## 1. 패키지 매니저/락파일 현황
- 확인된 락파일: `package-lock.json`
- 사용 매니저: **npm**
- 다른 매니저 관련 파일(`yarn.lock`, `pnpm-lock.yaml`)은 존재하지 않음 → 단일 npm 체계 유지 상태

## 2. “깨끗한 재설치” 절차 (lockfile 기준)
1. 설치된 모듈 제거  
   ```bash
   rm -rf node_modules package-lock.json
   ```
2. npm 캐시 초기화 (선택)  
   ```bash
   npm cache clean --force
   ```
3. 의존성 재설치  
   ```bash
   npm install
   ```

## 3. 의존성 점검 (삭제는 하지 않음)
| 항목 | 구분 | 비고 | 제안 |
| --- | --- | --- | --- |
| `clsx`, `tailwind-merge` | dependencies 버전 `*` | 잠재적 예측 불가 버전 사용 | 다음 단계에서 고정 버전 검토 |
| 전반 | deps vs devDeps 겹침 없음 | 현재 구분 오류 없음 | 유지 |
| 노이즈 패키지 | 미확인 | 불필요/미사용 후보 발견 불가 (수동 확인 필요) | 추후 사용처 검토 |

## 4. Node 엔진/타깃
- `package.json`에 이미 `engines: { "node": ">=20.0.0" }` 명시됨
- 내부 빌드/배포 환경과 맞는지 확인 후, 필요 시 README 문서에 동일 버전 명시만 권장 (수정은 미실행)

## 5. 요약
- 현재 npm 단일 사용 상태이며, lockfile 기반 깨끗한 재설치 방법을 문서화했음
- 버전 `*` 사용 패키지는 후속 PR에서 고정 버전 검토 필요
- 불필요 의존성은 추가 조사 후 오너 확인을 거쳐 삭제 추천 예정
